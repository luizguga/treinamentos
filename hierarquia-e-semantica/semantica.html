Semântica em HTML5

Tags Semânticas

Negrito e Itálico do jeito certo

Existem as tags <b> e <i> para essa tarefa, mas elas não possuem significado e focam apenas na forma, sendo assim, hoje em dia passou-se a usar as tags <strong> e <em> para realizar essas mesmas formatações visuais, só que agora com sentido.

A tag <strong> significa que o termo delimitado possui força dentro da frase. Logo, ele aparecerá em negrito. A tag <em> significa que queremos dar ênfase ao termo. Logo, ele aprecerá em itálico.

Texto marcado

Serve para marcar uma parte do texto na qual se quer dar uma ênfase descomunal, já que se trata de um trecho muito valioso para você.

Para fazer essas marcas em HTML5, usamos a tag <mark> para delimitar o texto que queremos demarcar.

Texto grande e pequeno

A tag <big> (que deixava o texto maior) está depreciada pela HTML5, mas a tag <small> (que deixa o texto menor) segue firme e forte na vida da linguagem.

Isso pode ser, porquê letras pequenas em contratos, por exemplo, tem um significado de observações mais técnicas do contrato. Já as letras grandes não possuem significado algum, somente forma, e forma, fica a cargo das CSS.

Texto deletado

Antigamente existia a tag <strike>, que hoje também está depreciada pela última versão da HTML. No lugar dela entrou a tag <del>. que significa que o texto ali, pode até ser lido, mas deve ser desconsiderado pelo leitor.

Texto inserido

O texto inserido é o exato oposto do texto deletado que vimos anteriormente. Nesse caso se colocarmos um texto qualquer dentro de <ins>, estamos dizendo que o texto está ali, deve ser lido e você deve prestar atenção nele. É o que fazíamos anteriormente ao utilizar a tag <u> para sublinhar um texto.

Texto Sobrescrito e Subscrito

Quantas vezes você tentou escrever um X2 ou um H2O em um documento e ficou na dúvida de como fazer para esses números ficarem pequeninos em cima ou embaixo. Para a HTML tem as tags <sub> (para subscrito) e <sup> (para sobrescrito).

Citações e códigos

Trechos de códigos e Pré-formatação

Nós somos programadores (ou seremos), e por isso compartilhamos muito código-fonte em diversas linguagens. Para isso, existe a tag <code> da HTML onde você pode delimitar seu código. A principal vantagem no uso dessa tag é o valor semântico que ela representa, indicando ao navegador que se trata de um código de computador. Porém, existe também um efeito visual, pois as letras ficam no modo mono-espaçadas (monospace), o que facilita bastante a leitura do código.

Você pode achar que o código vai aparecer formatado no seu site, porém o que acontece é uma simples mudança de letras. Para resolver esse problema, vamos usar uma outra tag HTML chamada <pre>, que mantém o texto pré-formatado, exatamente da mesma maneira na qual ele foi digitado, incluindo quebras de linhas, espaços e tabulações. Assim podemos usar as duas tags em combinação para exibir códigos em sites de maneira correta.

Citações

Normalmente, uma citação aparece entre aspas ou com as margens deslocadas em relação ao texto.

Para criar uma citação em HTML, podemos usar a tag <q> (do inglês quote, que significa citar). O texto que estiver entre <q> </q> já vai receber automaticamente as aspas, mas não terá nenhum deslocamento. Essa técnica é mais usada queremos uma citação no meio de um parágrafo.

Também podemos criar citações mais longas (em bloco) e que tenham um parágrafo só para si. Nesse caso, colocaremos tudo dentro de <blockquote> </blockquote> e o texto ganha um recuo automaticamente. Podemos também colocar um link para o texto original, usando o parâmetro cite dentro da tag.

Abreviações

Sempre que você quiser escrever uma sigla, mas deixar claro ao usuário (e aos mecanismos de busca) o significado dela, use a tag <abbr>. Use o parâmetro title para indicar o significado da sigla.
